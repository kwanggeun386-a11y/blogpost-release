# 다운로드 및 실행 확인 운영 방법

이 저장소는 GitHub가 기본 제공하는 다운로드 수 집계와 사용자 직접 확인 양식을 함께 사용합니다.

## 확인 가능한 것

- Release asset별 다운로드 횟수
- 저장소 방문 수와 clone 수
- 사용자가 직접 남긴 다운로드/실행 확인 Issue

## GitHub만으로 확인할 수 없는 것

- 누가 Release 파일을 다운로드했는지
- 다운로드한 사람이 실제로 실행했는지
- 다운로드한 사용자의 이메일, 회사명, PC 정보

GitHub는 일반 Release 다운로드에 대해 개인별 다운로드자 정보를 제공하지 않습니다.
따라서 사용자별 기록이 필요하면 확인 Issue를 남기도록 안내해야 합니다.

## 다운로드 수 확인

GitHub에서 직접 확인하려면:

1. 저장소의 `Actions` 탭으로 이동합니다.
2. `Release Download Report` 워크플로를 선택합니다.
3. `Run workflow`를 눌러 수동 실행합니다.
4. 실행이 끝나면 workflow run의 Summary에서 Release별 다운로드 수를 확인합니다.

이 워크플로는 매일 00:00 UTC에도 자동 실행됩니다.

## 사용자 확인 기록 받기

사용자에게 아래 링크를 안내합니다.

https://github.com/kwanggeun386-a11y/blogpost-release/issues/new?template=download-confirmation.yml

사용자는 다음 내용을 남깁니다.

- 다운로드한 버전
- 운영체제
- 다운로드한 파일
- 확인자 이름 또는 닉네임
- 실행 확인 상태
- 실행 실패 또는 특이사항

## 운영 팁

- 릴리즈 설명에 확인 Issue 링크를 항상 넣습니다.
- 사용자가 실행 실패를 남기면 Issue에서 바로 대응합니다.
- 공개 Issue이므로 개인정보, API 키, 비밀번호, 화면 캡처 속 민감정보를 올리지 말라고 안내합니다.
- 정확한 개인별 다운로드 강제가 필요하면 GitHub Release 직접 다운로드 대신 별도 다운로드 페이지와 로그인/폼 서버가 필요합니다.
